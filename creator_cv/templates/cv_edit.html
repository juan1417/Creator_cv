Contexto

Editar el JSON del contexto

Debe ser JSON válido y seguir el esquema del proyecto (mcp-ia-preguntas). Mantén las claves de nivel superior aunque algún bloque vaya vacío (listas [], strings ""). No se inventan datos: solo guarda lo que tú escribas.

Partes del JSON (referencia rápida)
  • meta — idioma, objetivo, tipo de CV, nombre, título, contacto.
  • perfil_profesional — solo resumen (texto). Opcional palabras_clave (legacy/ATS): en vista previa y exportación se muestran junto a técnicas, no bajo el perfil.
  • experiencia / educacion — listas de entradas con fechas y textos.
  • habilidades — técnicas, blandas, idiomas.
  • proyectos, certificaciones, fortalezas — según tu plantilla.
  • recursos_actuales — si pegas un CV previo o enlaces.
  • restricciones y dudas_pendientes — formato ATS, dudas sin resolver.
